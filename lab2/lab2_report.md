# 🧠 Описание интеграции

В рамках лабораторной работы был улучшен Telegram-бот StyleBot за счет интеграции с внешним источником данных — API погоды Open-Meteo.

Бот был доработан таким образом, чтобы получать актуальные данные о погоде по названию города и использовать их для формирования рекомендаций по одежде.

---

# 🌐 Источник данных

## Выбранный источник:
Open-Meteo API

- Geocoding API — для получения координат города
- Weather API — для получения текущей температуры

## Почему выбран:

- бесплатный API без необходимости регистрации
- простая структура запросов
- поддержка поиска городов по названию
- подходит для учебного проекта


# 📊 Структура данных

Ответ API приходит в формате JSON.

Пример:

{
  "current": {
    "temperature_2m": 7.7
  }
}

Используемые поля:
- latitude, longitude — координаты города  
- temperature_2m — текущая температура

Улучши моего Telegram-бота StyleBot, добавив интеграцию с API погоды Open-Meteo.

# ⚙️ Реализация

## Как работает интеграция

Пользователь вводит запрос:
- "погода в Москве"
- "что надеть в Париже"

Бот:
- извлекает название города  
- нормализует его (убирает падежи и лишние слова)  
- получает координаты через Geocoding API  
- запрашивает погоду через Weather API  
- формирует ответ

# 🧩 Ключевые фрагменты кода

## Нормализация города

Функция normalize_city_name приводит название города к более простой форме:
- убирает слова "завтра", "сегодня"
- обрабатывает падежи (например: казани → казань)

## Получение координат

Используется Geocoding API:
параметры: name (город), count=1, language=ru

## Получение погоды

Используется Weather API:
параметры: latitude, longitude, current=temperature_2m
