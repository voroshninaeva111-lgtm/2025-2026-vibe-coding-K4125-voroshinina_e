## 🚀 Описание деплоя

В рамках лабораторной работы был выполнен деплой Telegram-бота StyleBot в облачную среду.

В качестве способа деплоя была выбрана платформа **Railway**.

### Почему выбран Railway:
- простая интеграция с GitHub
- автоматический деплой при обновлении кода
- не требует настройки сервера
- подходит для учебных проектов

Бот успешно развернут и работает в режиме 24/7.

## ⚙️ Процесс деплоя

### Шаги деплоя:

1. Подготовлен проект:
   - проверена работа бота локально
   - создан файл `requirements.txt`
   - добавлен `.gitignore`
   - настроены переменные окружения (`BOT_TOKEN`)

2. Проект загружен на GitHub

3. На Railway:
   - создан новый проект
   - подключен GitHub репозиторий
   - добавлена переменная окружения `BOT_TOKEN`

4. Railway автоматически:
   - установил зависимости
   - запустил бота

---

### ❗ Проблемы и решения

**Проблема 1:**  
Ошибка при деплое (`SyntaxError: expected 'except' or 'finally' block`)

**Причина:**  
Неправильная структура `try/except` в коде

**Решение:**  
Исправлена вложенность блоков и добавлены недостающие `except`

---

**Проблема 2:**  
Git не видел изменения

**Причина:**  
Изменения в Cursor не были применены

**Решение:**  
Применены изменения через Cursor и сохранён файл
